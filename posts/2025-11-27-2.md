Svelte は“習得しやすく、実践的で高速”という特性を持ち、近年ますます注目されています。
気軽に小さなコンポーネントから書き始めて、フレームワークの自然さを体感してみてください。

## Svelte の最大の特徴：**「コンパイル型」フレームワーク**

Svelte が他のフレームワークと決定的に違うのは、**実行時ではなくビルド時にコンポーネントを最適化する点**です。

### 1. 仮想 DOM を使わない

React や Vue は “仮想 DOM” を使って差分を計算しますが、Svelte はこれを完全に排除。
コンパイル時に DOM 操作を最適化した命令へ変換するため、**高速かつ軽量なアプリ**を作れます。

### 2. ランタイムが極めて小さい

ランタイムがほぼ不要なため、バンドルサイズが劇的に小さくなります。
小さなプロジェクトから、大規模なアプリまでコストを抑えやすいのが魅力。

### 3. 文法がとにかくシンプル

Svelte はフレームワークというより **HTML・CSS・JavaScript の拡張**に近い感覚です。
余計な概念を覚える必要が少なく、学習コストが低いのも人気の理由です。

---

## Svelte コンポーネントの基本構造

Svelte コンポーネント（`.svelte` ファイル）は以下の 3 ブロックを基本とします：

```svelte
<script>
  let name = "Svelte";
</script>

<style>
  h1 {
    color: #ff3e00;
  }
</style>

<h1>Hello {name}!</h1>
```

HTML・CSS・JS が 1 ファイルにまとまっており、直感的に扱えるのが特徴です。

## まず覚えたい Svelte の文法

### ① 変数はそのまま表示できる（波括弧）

```svelte
<script>
  let count = 0;
</script>

<p>カウント: {count}</p>
```

---

### ② イベントは `on:` で書く

```svelte
<button on:click={() => count++}>
  増やす
</button>
```

React の `onClick` のようにキャメルケースではなく、`on:` で一貫しています。

---

### ③ リアクティブ宣言 `$:`

Svelte の最大の魅力とも言える機能。
変数が変わった瞬間に再評価されます。

```svelte
<script>
  let a = 1;
  let b = 2;

  $: sum = a + b;  // a または b が変わると自動更新
</script>

<p>{sum}</p>
```

依存関係を意識せず、直感的に反応するのが強力です。

---

### ④ if / else 文（テンプレート内制御）

```svelte
{#if loggedIn}
  <p>ようこそ</p>
{:else}
  <p>ログインしてください</p>
{/if}
```

---

### ⑤ each 文（配列ループ）

```svelte
<script>
  let items = ["りんご", "バナナ", "みかん"];
</script>

<ul>
  {#each items as item}
    <li>{item}</li>
  {/each}
</ul>
```

---

### ⑥ コンポーネントのプロパティ（props）

```svelte
<!-- Child.svelte -->
<script>
  export let title;
</script>

<h2>{title}</h2>
```

```svelte
<!-- 親 -->
<Child title="こんにちは" />
```

## Svelte が選ばれる理由まとめ

- 仮想 DOM なし → 高パフォーマンス
- 小さなランタイム → 軽量なバンドル
- 文法が HTML 直感的 → 学習がラク
- リアクティブ宣言が圧倒的に便利
- コンポーネント構造がシンプルで迷わない

特に「フロントエンドをこれから始める」人にとって扱いやすく、
「React などの大きなフレームワークより気軽に始めたい」場合にぴったりです。
